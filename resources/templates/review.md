# 리뷰 작성 가이드

> 공통 규칙은 `.claude/skills/meeting/notes-rules.md`를 먼저 적용하세요. 이 파일은 리뷰 형식의 본문 골격만 정의합니다.

---

## 작성 형식

회의록 본문 파일은 `$SESSION_DIR/meeting-notes.md`. 다음 골격을 따르세요:

```markdown
- 날짜: {날짜}
- 참석자: @{firstName1}, @{firstName2}, ...    ← 회사 멤버는 @firstName 접두 (publish 스킬 mention 변환). 외부 참석자는 평문, 매핑 미확정은 SPEAKER_XX
- 발표자: {매핑 확정 시 자연어 이름, 미확정이면 "(미확인)"}

## 개요
{리뷰 대상} — {리뷰 범위 1~2줄 설명}

- {scope_item1 — 피드백의 맥락을 잡아주는 최소한의 불릿}
- {scope_item2}

## 피드백

### ✅ 해소
- **SPEAKER_XX**: {피드백 내용}
  → {근거 포함 해소 내용}

### ❓ 미해소
- **SPEAKER_XX**: {피드백 내용} — {현재 상태/필요한 액션}

### 📌 추가 논의 필요
- {발표자가 열어둔 논의 사항 또는 논의 중 파생된 미결 건}

## 결정
- {decision} (없으면 "(없음)" 표기)

## 향후 일정
- {schedule}

## Action Items
- [ ] {task} @{담당자}    ← 1인칭이면 @SPEAKER_XX, 3인칭 지시면 산문 이름(@Bobs), 미정이면 역할(@백엔드)

## 참고 문서
- {reference}
```

---

## 작성 원칙

- **개요의 불릿**: 피드백 테이블을 읽을 때 맥락을 잡아주는 최소한의 범위 설명. **발표 내용을 재서술하지 마세요** — 원본 문서(PRD, 2-pager 등)가 있으므로 불필요
- **피드백**: 회의 중 제기된 피드백을 빠짐없이 기록. 해소 여부를 명확히 구분
  - **해소(✅)**: 회의 중 답변/설명으로 우려가 해소된 경우. `→`로 해소 내용을 들여쓰기. **근거와 전제 조건**까지 포함
  - **미해소(❓)**: 추가 검토/논의가 필요한 경우. `—`로 현재 상태를 인라인 표기
  - **여러 라인에 걸친 피드백은 핵심 1~2줄로 통합**: 한 SPEAKER가 7~10라인에 걸쳐 한 가지 의문을 말하면 핵심 의문만 1~2줄로 요약. 발화를 그대로 옮기지 말 것
- **추가 논의 필요(📌)**: 특정인의 피드백이 아닌, **발표자가 열어둔 논의 사항**이나 논의 중 파생된 미결 건. 피드백과 분리하여 성격이 다른 항목이 섞이지 않도록
- **결정**: 리뷰에서 결정이 나올 수 있음 (예: "이 스펙은 빼자"). 없으면 "(없음)"
- **Action Items**: 미해소/추가 논의에서 도출된 구체적 task만. 분기 규칙은 `.claude/skills/meeting/notes-rules.md` 참고

## 피드백 출처 처리

- 피드백 발화자의 SPEAKER_XX 라벨은 확신 있을 때만 박기 — 확신 없으면 이름 없이 불릿으로만 (예: `- 피드백 내용 → 해소 내용`, `**SPEAKER_XX**:` 접두사 생략)
- `_quality_warning: "severe_overmerge"` 세션은 `.claude/skills/meeting/notes-rules.md`의 품질 경고 처리 참고. 리뷰 유형은 발표자 70%+ 발화가 정상 패턴이라 경고가 false positive일 가능성 큼 → 경고 블록 삽입 불필요, 다만 피드백 출처는 보수적으로

## 예시 회의록

> 이 유형으로 만들어지는 회의록 샘플입니다 (형식 참고용 — 실제 작성 시 회의 내용으로 채웁니다).

```markdown
- 날짜: 2026-06-05
- 참석자: @Bobs, SPEAKER_02, SPEAKER_05
- 검토 대상: 신규 사용자 온보딩 화면 기획서

## 개요
- 첫 실행 시 3단계 온보딩 — 핵심 기능 소개 + 권한 요청 동선

## 피드백

### ✅ 해소
- **SPEAKER_02**: 권한 요청을 첫 화면에 몰면 이탈 우려
  → 권한은 해당 기능 진입 시점으로 분산, 첫 화면은 소개만

### ❓ 미해소
- **SPEAKER_05**: 건너뛰기 후 재진입 동선 누락 — 설정 메뉴 노출 여부 확정 필요

### 📌 추가 논의 필요
- 온보딩 완료율을 어느 지표까지 측정할지

## 결정
- 조건부 통과 — 건너뛰기 재진입 동선 보완 후 확정

## 향후 일정
- 다음 리뷰에서 수정안 재검토

## Action Items
- [ ] 건너뛰기 후 재진입 동선 추가 @Bobs
- [ ] 온보딩 완료율 지표 정의 @SPEAKER_05
```
