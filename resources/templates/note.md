# 회의록 작성 가이드 (note)

> 공통 규칙은 `.claude/skills/meeting/notes-rules.md`를 먼저 적용하세요. 이 파일은 회의록 형식의 본문 골격만 정의합니다.

---

## 작성 형식

회의록 본문 파일은 `$SESSION_DIR/meeting-notes.md`. 다음 골격을 따르세요:

```markdown
- 날짜: {날짜}
- 참석자: {이름1}, {이름2}, ...    ← 평문 이름. 매핑 미확정은 SPEAKER_XX

## 목표
- {goal}

## 논의

### 1. {topic}

- **SPEAKER_XX**: {발언 내용}
- **SPEAKER_XX**: {질문 또는 의견}
  - **→ SPEAKER_XX**: {응답}

> {note — 비고가 있는 경우}

### 2. {topic}
...

## 결정
- {decision}

## Action Items
- [ ] {task} @{담당자} (기한: {due})

## 관련 정보
- {reference}
```

---

## 사람별 보고 vs 주제별 논의 구분

### 순차 발표형 (Daily Scrum, 위클리 미팅 등 "진행자 호명 → 각자 진행 사항 발표" 패턴)

- **사람마다 별도 주제(`###`)로 분리하지 말 것** — 한 회의록에 8~10개 사람별 주제가 나오면 가독성 떨어짐
- **권장 1**: 의미 있는 작업/프로젝트 단위로 묶기
  - 예: `### 프로젝트 X` 안에서 `- **SPEAKER_03**: 1차 PR 머지` / `- **SPEAKER_07**: 리뷰 진행 중`
- **권장 2**: 묶을 단위가 마땅치 않으면 하나의 통합 주제(예: `### 주간 진행 보고`) 안에서 사람별 sub-section(`####`)으로 분리하거나, 그냥 SPEAKER 발언 불릿 나열
- **사람별 sub-section 헤더 표기** (권장 2의 sub-section 사용 시):
  - 매핑 확정: 자연어 이름만 — `#### Bobs`
  - 매핑 미확정: SPEAKER 라벨 + 역할/맥락 — `#### SPEAKER_06 (백엔드)` (사용자가 매핑 추가하면 자동 치환되어 자연스럽게 채워짐)
  - sub-section은 발언 단위가 아니라 "한 섹션 전체가 누구의 보고인지" 결정한 결과물이라, 매핑 확정 시 SPEAKER sentinel 박지 않음
- 진행자의 호명 멘트("다음 누구 발표 부탁드려요")는 회의록에 기록하지 않음 — 회의록 본문에 별도 "진행자" 줄도 작성하지 않음

### 자유 토론형 (비정기 논의, 브레인스토밍)

- 주제별로 분리. 각 주제 안에서 SPEAKER_XX 발언 나열

---

## 논의 포맷 규칙

- 각 논의 주제를 `###` 헤딩으로 구분
- 화자별 발언을 불릿(`-`)으로 나열, 화자명을 볼드(`**`)로 표기
- Q&A 흐름은 들여쓰기 + `→`로 응답 표시
- 비고는 인용문(`>`)으로 주제 아래에 추가
- 미확인 화자도 `SPEAKER_XX` 라벨 그대로 사용 (예: `**SPEAKER_08**: ...`). "참석자A" 같은 임시 이름 금지

---

## 결론·결정 처리

각 논의에 결론·상태 명시 (`.claude/skills/meeting/notes-rules.md`의 "결론 태그" 참고). 회의 전체의 핵심 결정은 별도 `## 결정` 섹션에도 정리.

---

## Action Items 처리

분기 규칙은 `.claude/skills/meeting/notes-rules.md`의 "Action Items `@assignee` 분기" 참고. 회의록 유형은 기한(due)이 명시되는 경우가 많으므로 `(기한: {due})` 형식 활용.

---

## 예시 회의록

> 이 유형으로 만들어지는 회의록 샘플입니다 (형식 참고용 — 실제 작성 시 회의 내용으로 채웁니다).

```markdown
- 날짜: 2026-06-05
- 참석자: Bobs, SPEAKER_03, SPEAKER_07

## 목표
- 주간 진행 상황 공유 및 일정 조정

## 논의

### 신규 기능 출시 준비
- **Bobs**: 1차 작업 완료, 다음 주 QA 예정
- **SPEAKER_03**: 일부 화면에서 예외 케이스 발견
  - **→ Bobs**: 별도 항목으로 분리해 처리하기로

### 고객 문의 대응 현황
- **SPEAKER_07**: 반복 문의 유형을 모아 FAQ 보강 (진행 중)

## 결정
- 출시 전 발견한 예외 케이스는 별도 항목으로 분리

## Action Items
- [ ] 예외 케이스 처리 항목 생성 @Bobs (기한: 6/9)
```
